## 1. ebook2audiobook

**GitHub:** [p0n1/epub_to_audiobook](https://github.com/p0n1/epub_to_audiobook)
**定位:** 多格式电子书转有声书工具

### 技术栈

| 组件 | 技术选型 |
|------|----------|
| 语言 | Python |
| TTS 引擎 | Azure TTS、OpenAI TTS、Edge TTS |
| 电子书解析 | ebooklib (EPUB)、BeautifulSoup |
| 音频处理 | pydub、ffmpeg |

### 功能特点

- **多格式支持:** EPUB、PDF、TXT 等格式输入
- **多 TTS 引擎:** 支持 Azure、OpenAI、Edge TTS 等多种引擎
- **章节分割:** 自动按章节生成独立音频文件
- **批量处理:** 支持整本书批量转换
- **命令行工具:** 面向技术用户的 CLI 界面

### 技术亮点

1. **模块化 TTS 引擎设计** - 易于扩展新的 TTS 提供商
2. **智能章节检测** - 基于 EPUB 结构自动分章
3. **断点续传** - 支持中断后继续处理

### 局限性

- 无角色识别功能，所有文本使用单一音色
- 无情绪/语调控制
- 仅支持命令行，无 Web 界面
- 不支持多角色自动分配

### SmartVoice 可借鉴

- 多 TTS 引擎的抽象层设计
- 章节分割和批量处理逻辑
- EPUB 解析方案

## 2. audiobook-creator

**GitHub:** [JarodMica/audiobook_maker](https://github.com/JarodMica/audiobook_maker)
**定位:** 智能多角色有声书生成器

### 技术栈

| 组件 | 技术选型 |
|------|----------|
| 语言 | Python |
| LLM | OpenAI GPT / 本地 LLM |
| TTS 引擎 | XTTS、StyleTTS2、RVC |
| 前端 | Gradio Web UI |
| 音频处理 | pydub、soundfile |

### 功能特点

- **LLM 角色识别:** 使用 GPT 分析文本，识别对话和角色
- **情感标签生成:** LLM 为每段对话生成情感标签
- **多 TTS 引擎支持:** XTTS、StyleTTS2、RVC 语音克隆
- **角色音色映射:** 支持为不同角色分配不同音色
- **Web 界面:** 基于 Gradio 的简易 Web UI

### 技术架构

```
文本输入 → LLM 分析 → 角色/情感标注 → TTS 生成 → 音频合并
              ↓
         角色识别
         情感分析
         对话分割
```

### 技术亮点

1. **LLM 驱动的角色识别** - 利用大语言模型理解文本语义
2. **情感标签系统** - 为 TTS 提供情感控制参数
3. **多 TTS 引擎集成** - 支持语音克隆技术
4. **角色一致性** - 同一角色保持相同音色

### 局限性

- 依赖 OpenAI API，成本较高
- 本地部署需要较高硬件配置（GPU）
- 角色识别准确率依赖 LLM 提示词质量
- 中文支持有限

### SmartVoice 可借鉴

- LLM 角色识别的 Prompt 设计思路
- 情感标签与 TTS 参数的映射方案
- 角色-音色绑定的数据结构设计

## 3. Coqui XTTS-v2

**GitHub:** [coqui-ai/TTS](https://github.com/coqui-ai/TTS)
**定位:** 多语言语音克隆模型

### 技术规格

| 指标 | 数值 |
|------|------|
| 模型大小 | ~1.5GB |
| 支持语言 | 17 种（含中文） |
| 克隆音频要求 | 6-30 秒参考音频 |
| 推理速度 | ~0.5x 实时（GPU） |
| 许可证 | CPML（非商业） |

### 功能特点

- **零样本语音克隆:** 仅需几秒参考音频即可克隆音色
- **多语言支持:** 支持中文、英文、日文等 17 种语言
- **跨语言克隆:** 可用 A 语言音频克隆后说 B 语言
- **情感控制:** 支持基本的情感和语调调节
- **流式生成:** 支持实时流式音频输出

### 技术架构

```
参考音频 → Speaker Encoder → Speaker Embedding
                                    ↓
文本输入 → Text Encoder → Decoder → 音频输出
```

### 技术亮点

1. **零样本克隆** - 无需训练即可克隆新音色
2. **多语言统一模型** - 单一模型支持多语言
3. **开源可部署** - 可本地部署，数据不出境

### 局限性

- CPML 许可证限制商业使用
- 中文效果不如英文
- 需要 GPU 才能达到实时速度
- 长文本生成可能出现音色漂移

### SmartVoice 可借鉴

- 语音克隆技术方案（用户自定义音色）
- 多语言支持架构
- Speaker Embedding 的存储和复用

## 4. Kokoro TTS

**GitHub:** [hexgrad/kokoro](https://github.com/hexgrad/kokoro)
**定位:** 轻量级开源 TTS 模型

### 技术规格

| 指标 | 数值 |
|------|------|
| 模型大小 | 82M 参数 |
| 支持语言 | 英语、中文、日语、韩语、法语 |
| 内置音色 | 54 种 |
| 推理速度 | CPU 可运行 |
| 许可证 | Apache 2.0 |

### 功能特点

- **轻量高效:** 82M 参数，CPU 即可运行
- **多语言支持:** 支持中英日韩法 5 种语言
- **丰富音色库:** 内置 54 种预设音色
- **商业友好:** Apache 2.0 许可证
- **易于集成:** 提供 Python API 和 ONNX 格式

### 技术亮点

1. **极致轻量** - 模型小，部署成本低
2. **商业可用** - Apache 2.0 许可证
3. **多平台支持** - 支持 CPU/GPU/移动端
4. **音色丰富** - 54 种预设音色覆盖多种场景

### 局限性

- 不支持语音克隆
- 情感控制能力有限
- 中文效果相对一般
- 音色定制需要微调训练

### SmartVoice 可借鉴

- 轻量模型的部署方案（降低成本）
- 预设音色库的设计思路
- 多语言支持的实现方式

## SmartVoice 技术启示

### 可直接借鉴

1. **audiobook-creator 的 LLM 角色识别方案**
   - 使用 LLM 分析文本结构
   - 提取对话、旁白、角色信息
   - 生成情感标签

2. **ebook2audiobook 的多引擎抽象层**
   - 统一的 TTS 接口设计
   - 易于切换和扩展 TTS 提供商

3. **Kokoro TTS 的轻量部署思路**
   - 预设音色库降低用户选择成本
   - CPU 可运行降低服务成本

### 需要自研或采购

1. **高质量中文 TTS**
   - 开源方案中文效果普遍一般
   - 建议采用讯飞/百度等商业 API

2. **角色识别模型**
   - 开源方案依赖通用 LLM
   - 可考虑训练专用的角色识别模型

3. **情感控制系统**
   - 需要与 TTS API 的情感参数对接
   - 建立情感标签到 TTS 参数的映射
