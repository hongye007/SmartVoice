**未来考虑：自建 TTS**
- 当用户量达到一定规模后
- 可考虑基于开源模型微调
- 降低长期成本

## 2. 角色识别技术对比

### 现有方案

| 方案 | 技术路线 | 准确率 | 成本 | 中文支持 |
|------|----------|--------|------|----------|
| audiobook-creator | GPT-4 Prompt | ~80% | 高 | 一般 |
| 规则匹配 | 正则表达式 | ~60% | 低 | 好 |
| NER 模型 | BERT-based | ~75% | 中 | 好 |
| 专用模型 | 微调 LLM | ~85% | 中 | 可优化 |

### SmartVoice 角色识别选型建议

**第一阶段（MVP）：混合方案**

```
文本输入 → 规则预处理 → LLM 精细识别 → 人工校正
              ↓              ↓
         引号检测        角色识别
         段落分割        情感分析
```

1. **规则预处理**
   - 引号检测（中文引号「」、""）
   - 对话标记识别（XX说、XX道）
   - 段落结构分析

2. **LLM 精细识别**
   - 使用 GPT-4 或国产大模型（如通义千问）
   - 识别角色名称和对话归属
   - 生成情感标签

3. **人工校正**
   - 提供校正界面
   - 用户可修正识别错误
   - 收集数据用于模型优化

**第二阶段（成熟期）：专用模型**

- 基于收集的数据训练专用模型
- 降低对 LLM API 的依赖
- 提高准确率和降低成本

### SmartVoice 语言策略

**MVP 阶段：中文优先**
- 主要服务中文用户
- 采用讯飞 TTS 确保中文质量
- 英文作为次要支持

**成长阶段：多语言扩展**
- 根据用户需求扩展语言
- 可接入多个 TTS 引擎
- 不同语言使用最优引擎

## 4. 部署方式对比

| 方案 | 部署方式 | 成本 | 扩展性 | 数据安全 |
|------|----------|------|--------|----------|
| 纯云端 API | SaaS | 按量付费 | 高 | 依赖第三方 |
| 混合部署 | 云+本地 | 中等 | 中 | 较好 |
| 完全自建 | 私有化 | 高 | 低 | 最好 |

### SmartVoice 部署建议

**MVP 阶段：纯云端**
- 使用讯飞/百度 TTS API
- 阿里云 OSS 存储音频
- 快速上线，低初始成本

**成长阶段：混合部署**
- 核心逻辑自建
- TTS 仍使用 API
- 考虑成本优化

## 6. SmartVoice 技术选型总结

### 推荐技术栈

| 组件 | 选型 | 理由 |
|------|------|------|
| **前端** | React + TypeScript + Tailwind | 现代化、组件丰富、开发效率高 |
| **后端** | Node.js (Express/Fastify) | 与前端技术栈统一，异步处理能力强 |
| **TTS 引擎** | 讯飞 TTS API（主）+ 百度（备） | 中文效果最好，价格合理 |
| **角色识别** | 规则 + LLM（通义千问/GPT） | 混合方案，平衡准确率和成本 |
| **存储** | 阿里云 OSS | 国内访问快，价格合理 |
| **数据库** | PostgreSQL | 功能强大，JSON 支持好 |
| **缓存** | Redis | 会话管理，任务队列 |

## 7. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| TTS API 涨价 | 成本上升 | 多引擎备选，长期考虑自建 |
| 角色识别准确率不足 | 用户体验差 | 提供手动校正，持续优化 |
| LLM API 不稳定 | 服务中断 | 多 LLM 备选，本地缓存 |
| 中文效果不达预期 | 用户流失 | 与讯飞深度合作，定制优化 |
