## 🎯 用户背景

### 工作/生活环境

林晓是一名网络小说作家,同时也尝试制作有声版作品增加收入。她白天在家创作,晚上处理粉丝互动和内容发布。创作的小说类型包括言情、玄幻、悬疑等,通常有多个角色和复杂的对话场景。

**典型的一天:**
- 上午创作新章节(3000-10000字)
- 下午修改润色,准备发布
- 晚上想将完成的章节制作成有声版,但现有工具无法满足需求
- 使用 SmartVoice 的场景:完成章节创作后,上传文本,预览角色配音效果,微调音色,分章节生成有声内容

### 行为特征

- **技术使用习惯:** 主要使用电脑创作,手机查看数据和互动,熟悉文字处理软件,对音频编辑工具不够熟悉
- **信息获取方式:** 通过作者社群、创作平台、短视频平台学习新工具,重视其他创作者的推荐
- **购买决策模式:** 价格敏感但愿意为提升效率付费,通常先试用再购买,关注性价比
- **社交习惯:** 活跃在作者群、豆瓣、知乎、微博,与粉丝和同行交流创作心得

## 🎯 目标与动机

### 主要目标

**核心目标:**
将创作的文字内容快速转化为高质量多角色有声作品,增加作品传播渠道和收入来源,提升内容竞争力。

**次要目标:**
1. 降低有声内容制作成本(相比雇佣声优)
2. 缩短制作周期,能够与文字更新同步
3. 保持角色音色的一致性和专业性
4. 提升粉丝满意度和作品影响力

### 动机

**为什么他们想要达成这些目标?**
- **内在动机:** 希望作品以更丰富的形式呈现,实现创作价值最大化,获得成就感
- **外在动机:** 增加收入渠道,在竞争激烈的内容市场中脱颖而出,满足粉丝需求

## 😞 痛点与挫折

### 当前面临的主要痛点

**痛点 1:有声内容制作成本高**
- **描述:** 雇佣声优录音费用高昂(单章数百至数千元),对于更新频繁的连载作品难以承受
- **影响:** 无法为所有作品制作有声版,限制了作品传播和收入增长
- **频率:** 每周或每天(取决于更新频率)
- **严重程度:** 🔴 高

**痛点 2:现有TTS工具无法满足多角色需求**
- **描述:** 讯飞、微软等TTS工具音色单一,无法智能识别和分配角色,不支持情感表达,生成的音频缺乏生动性
- **影响:** 制作出的有声内容质量差,角色难以区分,粉丝不买账
- **频率:** 每次尝试使用TTS工具时
- **严重程度:** 🔴 高

**痛点 3:制作效率低、流程复杂**
- **描述:** 需要手动分段、标注角色、逐个录音或配置,一章内容可能需要数小时处理
- **影响:** 严重影响创作时间,无法保证与文字更新同步,导致放弃制作有声版
- **频率:** 每次制作有声内容
- **严重程度:** 🟡 中

### 挫折来源

- **工具/流程相关:** 现有工具功能单一,不理解创作场景,需要在多个工具间切换
- **时间相关:** 有声内容制作耗时过长,挤占创作时间
- **复杂性相关:** 音频处理技术门槛高,需要学习专业知识
- **质量相关:** 即使投入大量时间,效果仍不理想,难以达到专业水准

## 🔧 当前解决方案

### 他们目前如何解决问题?

**主要工具/方法:**
1. **雇佣声优录音**
   - 用途:为重点作品制作专业有声版
   - 优点:质量好,角色区分度高
   - 缺点:成本高昂,周期长,只能用于少数作品

2. **听书App平台(如喜马拉雅、懒人听书)**
   - 用途:上传作品,依赖平台用户朗读
   - 优点:无需自己录制,有一定收入
   - 缺点:内容质量参差不齐,分成少,依赖他人,无法自主控制

3. **系统TTS工具(如讯飞、微软)**
   - 用途:尝试快速生成有声版
   - 优点:成本低,速度快
   - 缺点:音色单一,无法区分角色,缺乏情感,质量差

### 工作流程

当前制作有声内容的流程:

```
创作文字 → 寻找声优/平台 → 沟通需求 → 等待录制 → 审核修改 → 发布
(或)
创作文字 → 手动标注角色 → 逐段使用TTS → 音频拼接 → 效果不理想 → 放弃
```

**痛点所在:**
- 在"寻找声优"步骤,面临成本和周期压力
- 在"使用TTS"步骤,面临质量和效率问题
- 整体流程导致大部分作品无法制作有声版

## 💡 对产品的需求

### 必须有的功能 (Must-Have)

1. **智能角色识别与自动分配**
   - 原因:多角色是核心需求,手动标注太耗时
   - 期望结果:上传文本后自动识别对话、旁白和角色,自动分配合适音色

2. **音色预览和微调**
   - 原因:需要确保音色符合角色设定,避免返工
   - 期望结果:快速预览效果,能够调整音色、音速、情绪等参数

3. **分章节批量生成**
   - 原因:作品通常有多个章节,需要高效处理
   - 期望结果:支持单章预览微调,确认后批量生成所有章节,保持角色音色一致

4. **高质量音频输出**
   - 原因:音质直接影响用户体验和作品价值
   - 期望结果:接近真人水准的音色和情感表达,清晰流畅

### 希望有的功能 (Nice-to-Have)

- **情绪和语调智能理解**:根据文本语境自动调整情绪(如愤怒、悲伤、喜悦)
- **背景音效和配乐**:为作品增加氛围感
- **多格式导出**:支持 MP3、M4A 等主流格式
- **历史记录和模板保存**:同一作品的后续章节可复用角色设置

### 不需要的功能

- **实时语音合成API**:不需要集成到其他系统,Web应用即可
- **多语言支持**:主要创作中文内容,其他语言需求不强烈
- **协作功能**:通常独立创作,暂时不需要多人协作

- **质量提升**:"粉丝反馈有声版质量好,角色区分明显,愿意为此付费"

## 🚫 使用障碍

### 可能阻碍用户采用产品的因素

**技术障碍:**
- **AI识别准确度担忧**:"担心AI识别错误,需要大量手动修正"
- **音色质量疑虑**:"担心AI音色不够自然,效果不如真人"

**心理障碍:**
- **学习成本顾虑**:"担心新工具太复杂,学习耗时"
- **效果不确定性**:"不确定能否真正达到理想效果"

**经济障碍:**
- **价格敏感**:"如果价格过高,还不如继续使用免费的听书平台"

### 如何克服这些障碍?

- **针对准确度担忧:**提供免费试用,让用户亲自体验识别准确率(目标>85%)
- **针对质量疑虑:**提供真实案例展示,用户盲测对比
- **针对学习成本:**设计简单直观的界面,"一键生成"降低上手门槛,提供新手引导
- **针对价格敏感:**提供免费额度或低价入门套餐,按字数/时长灵活计费

## 📈 画像使用指南

### 如何使用这个画像?

**在产品设计时:**
- 确保核心流程符合林晓的创作习惯:创作完成 → 快速上传 → 智能生成 → 微调 → 导出
- 界面设计要简单直观,避免专业术语,降低学习成本

**在优先级决策时:**
- 优先实现智能角色识别、音色定制、批量生成这些核心需求
- 次要功能(如背景音乐)可在MVP后迭代

**在营销推广时:**
- 在作者社群、内容创作平台(如起点、晋江、豆瓣)进行推广
- 强调"成本低、效率高、质量好"的核心卖点
- 使用真实案例和对比效果吸引用户
