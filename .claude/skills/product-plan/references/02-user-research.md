## 👤 用户画像开发

### 什么是用户画像？

用户画像 (User Persona) 是基于真实用户研究创建的虚构代表性用户。它帮助团队保持对目标用户的共同理解。

### 创建用户画像的步骤

**1. 收集数据**
- 用户访谈 (10-20 人)
- 用户调研
- 数据分析
- 客户支持反馈

**2. 识别模式**
- 相似的行为模式
- 共同的目标和动机
- 类似的痛点和挫折
- 相近的人口统计特征

**3. 创建画像**
- 给画像起名字
- 添加背景故事
- 描述目标和动机
- 列出痛点和需求

**4. 验证和迭代**
- 与真实用户验证
- 团队共识
- 定期更新

### 用户画像包含的要素

**基本信息：**
- 姓名和照片
- 年龄、职业、背景
- 技术熟练度

**行为特征：**
- 典型的一天
- 使用的工具和平台
- 决策方式

**目标与动机：**
- 主要目标
- 次要目标
- 深层动机

**痛点与挫折：**
- 当前面临的问题
- 使用现有解决方案的不满
- 未被满足的需求

**对产品的期望：**
- 必须有的功能
- 希望有的功能
- 成功标准

## 🎯 Jobs-to-be-Done 框架

### JTBD 理论

用户"雇用"产品是为了完成特定的"工作"。理解用户要完成的工作比理解用户特征更重要。

### JTBD 陈述格式

```
当我 [情境]，
我想要 [动机]，
以便我能 [期望结果]。
```

**示例：**
> 当我准备与重要客户开会时，我想要快速了解我们过去所有的互动历史，以便我能提供个性化的服务并赢得信任。

### JTBD 的三个维度

**1. 功能性工作 (Functional Job)**
- 用户想要完成的实际任务
- 示例："我想要分析销售数据"

**2. 情感性工作 (Emotional Job)**
- 用户想要的感受
- 示例："我想要感到自信和专业"

**3. 社会性工作 (Social Job)**
- 用户想要的社会形象
- 示例："我想要被团队视为高效的管理者"

### 发现 JTBD 的方法

**访谈问题：**
- "你为什么开始使用这个产品？"
- "在什么情况下你会使用它？"
- "使用前后有什么变化？"
- "如果这个产品不存在，你会怎么做？"

## 😞 痛点识别

### 痛点类型

**1. 流程痛点**
- 当前流程繁琐、耗时
- 示例："我需要在 5 个工具间切换才能完成一个任务"

**2. 财务痛点**
- 成本过高、性价比低
- 示例："现有解决方案太贵了"

**3. 生产力痛点**
- 效率低下、浪费时间
- 示例："手动数据录入占用我 30% 的工作时间"

**4. 支持痛点**
- 缺乏帮助、学习困难
- 示例："遇到问题时找不到答案"

### 痛点优先级评估

**评估维度：**
1. **严重性** - 问题有多糟糕？
   - 阻碍工作 (10分)
   - 降低效率 (5分)
   - 轻微不便 (1分)

2. **频率** - 多久发生一次？
   - 每天 (10分)
   - 每周 (5分)
   - 偶尔 (1分)

3. **影响范围** - 影响多少用户？
   - 所有用户 (10分)
   - 大部分用户 (5分)
   - 少数用户 (1分)

**优先级 = 严重性 × 频率 × 影响范围**

只解决高优先级痛点。

## 💬 用户访谈

### 访谈准备

**确定目标：**
- 你想要了解什么？
- 哪些假设需要验证？

**招募用户：**
- 目标用户画像
- 5-10 人即可发现主要模式
- 提供适当激励

**准备问题：**
- 开放式问题
- 避免引导性问题
- 准备 10-15 个核心问题

### 访谈技巧

**1. 建立信任**
- 解释访谈目的
- 保证隐私
- 营造轻松氛围

**2. 提问技巧**
✅ "跟我讲讲你上次使用 X 的经历"
❌ "你喜欢 X 功能吗？"

✅ "什么情况下你会使用这个工具？"
❌ "你会使用我们计划的这个功能吗？"

**3. 深入挖掘**
- 使用 5 个为什么
- "能详细说说吗？"
- "还有呢？"

**4. 观察行为**
- 让用户演示
- 观察实际行为
- 记录非语言信号

### 访谈问题库

**背景问题：**
- 能否介绍一下你的工作/角色？
- 你的典型工作日是怎样的？

**行为问题：**
- 你现在如何完成 [任务]？
- 上次使用 [工具] 是什么时候？
- 能带我走一遍你的流程吗？

**痛点问题：**
- 这个过程中最让你沮丧的是什么？
- 什么事情占用你最多时间？
- 如果有魔法棒，你最想改变什么？

**需求问题：**
- 理想情况下，这个任务应该如何完成？
- 什么功能会让你的工作更轻松？
- 你愿意为解决这个问题付出什么？

## 🗺️ 用户旅程地图

### 什么是用户旅程地图？

用户旅程地图展示用户与产品交互的完整流程，包括每个阶段的行为、想法、情感和痛点。

### 旅程阶段

**1. 意识阶段 (Awareness)**
- 用户如何发现问题？
- 如何了解到产品？

**2. 考虑阶段 (Consideration)**
- 用户如何评估解决方案？
- 比较哪些选项？

**3. 购买/注册阶段 (Purchase/Signup)**
- 决策过程如何？
- 有哪些障碍？

**4. 使用阶段 (Usage)**
- 首次使用体验
- 日常使用流程
- 高级功能探索

**5. 忠诚/推荐阶段 (Loyalty/Advocacy)**
- 成为重度用户
- 向他人推荐

### 旅程地图模板

| 阶段 | 用户行为 | 想法 | 情感 | 痛点 | 机会点 |
|------|---------|------|------|------|--------|
| 意识 | [行为] | [想法] | 😊/😐/😞 | [痛点] | [机会] |
| 考虑 | [行为] | [想法] | 😊/😐/😞 | [痛点] | [机会] |
| 注册 | [行为] | [想法] | 😊/😐/😞 | [痛点] | [机会] |
| 使用 | [行为] | [想法] | 😊/😐/😞 | [痛点] | [机会] |
| 忠诚 | [行为] | [想法] | 😊/😐/😞 | [痛点] | [机会] |

### 绘制旅程地图的步骤

1. **选择焦点画像**
2. **定义场景和目标**
3. **列出所有接触点**
4. **填充每个阶段的详细信息**
5. **识别痛点和机会**
6. **可视化情感曲线**
7. **提出改进建议**

## 📊 用户研究方法对比

| 方法 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **用户访谈** | 深度理解动机和痛点 | 深入洞察 | 样本小 |
| **问卷调查** | 量化验证假设 | 大规模数据 | 深度不足 |
| **可用性测试** | 评估产品易用性 | 发现实际问题 | 需要原型 |
| **A/B 测试** | 优化具体设计 | 数据驱动 | 只能测试存在的 |
| **数据分析** | 了解实际行为 | 真实客观 | 不知道"为什么" |

**建议组合使用：**
- 定性研究 (访谈、观察) + 定量研究 (调研、数据)
- 探索性研究 (早期) + 验证性研究 (后期)

## 🎯 实践检查清单

### 用户研究准备

- [ ] 明确研究目标和问题
- [ ] 确定目标用户群体
- [ ] 准备访谈/调研问题
- [ ] 招募合适的用户
- [ ] 准备记录工具

### 执行研究

- [ ] 进行用户访谈 (5-10人)
- [ ] 观察用户实际行为
- [ ] 记录详细笔记和录音
- [ ] 收集量化数据
- [ ] 拍照记录(如适用)

### 分析和综合

- [ ] 整理所有数据
- [ ] 识别行为模式
- [ ] 提取关键洞察
- [ ] 创建用户画像
- [ ] 绘制用户旅程地图

### 应用到产品

- [ ] 与团队分享研究结果
- [ ] 更新产品需求
- [ ] 优先级排序
- [ ] 定义成功指标
- [ ] 计划持续研究

---

**记住：** 用户研究不是一次性活动，而是持续的过程。随着产品发展和用户变化，定期重访和更新用户理解。
