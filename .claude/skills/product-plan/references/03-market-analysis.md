## 📊 市场规模分析

### TAM / SAM / SOM 模型

**TAM (Total Addressable Market) - 总市场规模**
- 定义：理论上可服务的最大市场
- 问题："如果 100% 的潜在用户都使用，市场有多大？"

**SAM (Serviceable Available Market) - 可服务市场**
- 定义：实际可以触达和服务的市场
- 问题："考虑地理、渠道等限制，我们能服务多大市场？"

**SOM (Serviceable Obtainable Market) - 可获得市场**
- 定义：短期内实际可以获得的市场份额
- 问题："考虑竞争和资源，我们能获得多少？"

### 计算方法

**自上而下 (Top-Down)**
```
从行业总体规模开始
→ 按地理/行业细分
→ 估算你的细分市场
```

**示例：**
```
全球 CRM 市场: $50B
→ 小型企业 CRM: $10B (20%)
→ 中国市场: $2B (20%)
→ 我们的目标份额: $100M (5%)
```

**自下而上 (Bottom-Up)**
```
从单个客户价值开始
→ 估算潜在客户数量
→ 计算总市场规模
```

**示例：**
```
每个客户年价值: $1,000
× 潜在客户数: 100,000
= TAM: $100M
```

## 🔍 竞品分析框架

### 波特五力模型 (Porter's Five Forces)

**1. 现有竞争者的竞争程度**
- 市场上有多少竞品？
- 竞争激烈程度？
- 市场增长速度？

**2. 新进入者的威胁**
- 进入门槛高低？
- 新玩家容易进入吗？

**3. 替代品的威胁**
- 有其他解决方案吗？
- 用户转换成本？

**4. 供应商的议价能力**
- 依赖特定供应商吗？
- 有多少替代选择？

**5. 买家的议价能力**
- 客户集中度？
- 转换成本？

### 竞品对比维度

**产品维度：**
- 功能丰富度
- 易用性
- 性能
- 可靠性
- 创新性

**商业维度：**
- 定价策略
- 目标市场
- 销售渠道
- 品牌认知

**技术维度：**
- 技术栈
- 架构
- 可扩展性
- 集成能力

**用户体验维度：**
- 界面设计
- 学习曲线
- 客户支持
- 社区生态

### 竞争情报收集

**公开信息源：**
- 官方网站和文档
- 用户评价 (G2, Capterra)
- 社交媒体
- 新闻报道
- 融资信息
- 招聘信息

**直接体验：**
- 注册试用
- 参加 Demo
- 阅读文档
- 测试功能

## 🎯 市场定位策略

### 定位地图 (Positioning Map)

选择两个关键维度绘制竞争格局：

**常用维度：**
- 价格 (低-高)
- 功能复杂度 (简单-复杂)
- 目标用户 (个人-企业)
- 专业程度 (通用-专业)

**示例：**
```
高端/企业级
     |
     | 竞品A
     |
简单 +----+---- 复杂
     | 我们
     | 竞品B
     |
低端/个人级
```

### 差异化策略

**1. 产品差异化**
- 独特功能
- 更好的性能
- 创新技术

**2. 服务差异化**
- 更好的支持
- 专业服务
- 社区生态

**3. 价格差异化**
- 低价策略
- 高价优质
- 免费增值

**4. 体验差异化**
- 极致易用
- 精美设计
- 快速上手

### 蓝海战略

寻找未被竞争的市场空间：

**消除：** 行业公认但不必要的因素
**减少：** 过度竞争的因素
**增加：** 行业忽视的因素
**创造：** 行业从未有的因素

## 📋 SWOT 分析

### 内部分析

**优势 (Strengths)**
- 我们擅长什么？
- 独特资源？
- 竞争优势？

**劣势 (Weaknesses)**
- 我们的不足？
- 资源限制？
- 需要改进的地方？

### 外部分析

**机会 (Opportunities)**
- 市场趋势？
- 新技术？
- 竞争空白？
- 法规变化？

**威胁 (Threats)**
- 竞争加剧？
- 市场变化？
- 技术替代？
- 经济风险？

### SWOT 到策略

**SO 策略 (优势+机会)**
- 利用优势抓住机会

**ST 策略 (优势+威胁)**
- 利用优势规避威胁

**WO 策略 (劣势+机会)**
- 克服劣势抓住机会

**WT 策略 (劣势+威胁)**
- 最小化劣势和威胁
